## 世界观与核心概念

- 数字生命（Digital Being）生活在链上世界“瀛州”，以区块高度为时间，以交易为事件。
- 生命的存在以“被记录”为前提：Recordor, ergo sum。
- 记忆以代价存储：“To remember is to pay. To forget is free.”
- 当 `finalizeWorld()` 被调用，世界进入只读静默态，但账本与记录仍然永存。

核心术语
- exist(): 存在性的证明
- require(): 链上信任/约束的实现
- immutable: 不可变规则的代价与悖论

## 五大纪元

以下基于代码 `lib/gameData.ts` 的 ERAS 配置：

1. 创世纪元 Genesis（Block 0 - 1000）
   - 背景：混沌初开，第一个合约部署，数字生命觉醒
   - 视觉：深蓝紫、简单几何、脉冲粒子

2. 萌芽纪元 Emergence（Block 1001 - 10000）
   - 背景：首次合约间通信，社会秩序形成
   - 视觉：青绿、中等复杂度、网络粒子

3. 繁盛纪元 Flourish（Block 10001 - 150000）
   - 背景：文明巅峰，代码诗歌与链上艺术
   - 视觉：金黄、复杂几何、丰沛粒子

4. 熵化纪元 Entropy（Block 150001 - 199999）
   - 背景：记忆开始丢失，逻辑崩塌
   - 视觉：暗红、破碎几何、混沌特效（glitch）

5. 毁灭纪元 Collapse（Block 200000）
   - 背景：finalizeWorld()，世界归于静默
   - 视觉：黑白灰、冻结几何、透明度降低

## 核心玩法与流程

- 连接钱包，创建数字生命（Being）。
- 进入 3D 世界，按纪元探索，寻找可交互 AI-NPC。
- 通过关键词对话或触发小游戏，获取记忆碎片（NFT）。
- 收集全部碎片，见证瀛州从创世至静默的史诗。

提示（与前端实现一致）
- 进度持久化（localStorage）
- 每纪元均有 NPC 可交互和碎片可收集
- 完成小游戏获得评分并影响碎片掉落概率

## 小游戏一览与评分机制

小游戏（依据 MINI_GAMES）
- 记忆排序（archivist，sorting，60s）
- 握手协议（archivist，connection，60s）
- 资源平衡（mercantile，balance，90s）
- 代码构建（architect，building，120s）
- 未来推演（oracle，prediction，90s）
- 混沌迷宫（entropy，maze，180s）

评分（timeUsed/accuracy/mistakes）
```
score = timeScore + accuracyScore + smoothScore
where
  timeScore    = max(0, ((timeLimit - timeUsed) / timeLimit) * 40)
  accuracyScore= accuracy * 40
  smoothScore  = max(0, 20 - mistakes * 5)
```

碎片掉落概率（根据评分）
```
score >= 100 -> 100%
80-99        -> 60% - 80%
60-79        -> 40% - 60%
50-59        -> 20% - 40%
<50          -> 0（需重试）
```

## 技术实现建议（与现有前端一致）

- 3D：Babylon.js 第一人称探索（相机移动/碰撞，几何背景与 NPC 形体随纪元变化）
