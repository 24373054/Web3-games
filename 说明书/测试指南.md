## ? 如何测试

### 步骤1：启动项目

```bash
# 1. 进入项目目录
cd Web3-games-main/瀛州纪

# 2. 安装依赖（如果还没安装）
npm install

# 3. 编译合约
npm run compile

# 4. 启动本地区块链（新终端）
npx hardhat node

# 5. 部署合约（新终端）
npm run deploy
# 复制输出的合约地址

# 6. 配置环境变量
# 创建 .env.local 文件，填入合约地址
NEXT_PUBLIC_CHAIN_ID=31337
NEXT_PUBLIC_RPC_URL=http://127.0.0.1:8545
NEXT_PUBLIC_WORLD_LEDGER_ADDRESS=你的WorldLedger地址
NEXT_PUBLIC_DIGITAL_BEING_ADDRESS=你的DigitalBeing地址
NEXT_PUBLIC_AINPC_ADDRESS=你的AINPC地址

# 7. 启动前端
npm run dev

# 8. 访问 http://localhost:3000
```

## ? 测试技巧

### 快速完成任务
如果想快速测试，可以输入包含多个关键词的问题，例如：

**创世纪元一次性测试：**
```
"告诉我瀛州的创世历史、底层架构、以及数字生命如何证明自己的存在"
```

这会触发多个关键词匹配，可能一次完成多个任务。

### 查看控制台日志
打开浏览器开发者工具（F12），查看控制台：
- 任务检测信息
- 关键词匹配结果
- 进度保存日志

### 手动编辑进度
如果想跳到特定纪元测试，可以：
1. F12打开控制台
2. 输入：
```javascript
localStorage.setItem('yingzhou_progress', JSON.stringify({
  completedQuests: [],
  currentEra: 3, // 改成你想测试的纪元 (0-4)
  unlockedContent: [],
  narrativeFragments: {}
}))
location.reload()
```

### 重置进度
```javascript
localStorage.removeItem('yingzhou_progress')
location.reload()
```

## ? 常见问题排查

### 问题1：任务没有完成
**原因：**
- 关键词不匹配
- NPC类型不对

**解决：**
- 查看任务提示中的关键词
- 确保选择了正确的NPC
- 尝试不同的提问方式

### 问题2：历史碎片没有显示
**原因：**
- 任务未真正完成
- 前端状态未更新

**解决：**
- 检查任务列表是否显示?
- 刷新页面
- 查看localStorage中的进度

### 问题3：纪元没有推进
**原因：**
- 还有未完成的任务
- 进度没有保存

**解决：**
- 确认所有3个任务都完成（100%）
- 刷新页面让状态更新
- 手动检查completedQuests数组

### 问题4：进度丢失
**原因：**
- localStorage被清除
- 浏览器隐私模式

**解决：**
- 不要使用隐私/无痕模式
- 不要清除浏览器数据
- 每次完成任务后刷新确认保存
