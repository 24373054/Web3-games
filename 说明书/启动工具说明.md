## ? 快速启动

### 方式一: 一键启动 (推荐)

**Windows**:
```cmd
start.bat
```

**macOS/Linux**:
```bash
./start.sh
```

**跨平台**:
```bash
npm run quick-start
```

### 方式二: 手动启动

```bash
# 1. 安装依赖
npm install

# 2. 编译合约
npm run compile

# 3. 启动本地区块链 (新终端)
npx hardhat node

# 4. 部署合约 (新终端)
npm run deploy

# 5. 启动开发服务器
npm run dev
```

## ?? 启动工具

### 1. `start-dev.js` - 自动化启动脚本

**功能**:
- ? 环境检测 (Node.js, npm, Git)
- ? 自动安装依赖
- ? 创建环境配置文件
- ? 编译智能合约
- ? 启动本地区块链
- ? 部署智能合约
- ? 启动开发服务器

**使用**:
```bash
node start-dev.js
# 或
npm run quick-start
```

**特点**:
- ? 一键完成所有准备工作
- ? 自动检测已运行的服务
- ? 彩色输出,清晰易读
- ? 并行启动多个服务
- ?? 错误处理和提示

**适用场景**:
- 首次克隆项目
- 长时间未开发后重新启动
- 环境配置不确定时
- 新手开发者

**功能**:
- 检查 Node.js 和 npm
- 调用 `start-dev.js`
- Windows 友好界面

**使用**:
```cmd
双击 start.bat
或
.\start.bat
```

**特点**:
- ?? 双击即可运行
- ? 无需打开命令行
- ? 中文界面

## ? 诊断工具

### `diagnose.js` - 环境诊断脚本

**功能**:
全面检查开发环境和项目配置

**检查项目**:

#### 1. 系统环境
- ? 操作系统类型
- ? Node.js 版本
- ? npm 版本
- ? Git 可用性
- ? 系统内存
- ? CPU 核心数

#### 2. 项目结构
- ? 必需文件 (package.json, tsconfig.json 等)
- ? 必需目录 (app/, components/, lib/ 等)

#### 3. 依赖安装
- ? node_modules 存在性
- ? 关键依赖完整性
- ? 依赖版本信息

#### 4. 环境配置
- ? .env 文件存在性
- ? 环境变量配置

#### 5. 智能合约
- ? contracts/ 目录
- ? 合约文件数量
- ? 编译状态
- ? 合约地址配置

#### 6. 游戏数据
- ? 核心游戏文件
- ? 组件完整性
- ? 小游戏实现

#### 7. 端口检查
- ? 3000 端口 (Next.js)
- ? 8545 端口 (Hardhat)

**使用**:
```bash
node diagnose.js
# 或
npm run diagnose
npm run check
```

**输出**:
- ? 彩色控制台输出
- ? 分类统计 (通过/警告/失败)
- ? 针对性建议
- ? 生成 `diagnostic-report.txt`

**使用场景**:
- 启动失败时
- 环境迁移后
- 依赖更新后
- 寻求帮助前

**示例输出**:

## ? 常见使用场景

### 场景 1: 首次启动项目

```bash
# 1. 克隆项目
git clone <repository-url>
cd 瀛州纪

# 2. 一键启动
npm run quick-start

# 或使用便捷脚本
./start.sh  # macOS/Linux
start.bat   # Windows
```

### 场景 2: 环境出现问题

```bash
# 1. 运行诊断
npm run diagnose

# 2. 查看报告
cat diagnostic-report.txt

# 3. 根据建议修复
npm install  # 如果缺少依赖

# 4. 重新启动
npm run quick-start
```

### 场景 3: 更新依赖后

```bash
# 1. 清理环境
npm run clean

# 2. 重新安装
npm install

# 3. 诊断检查
npm run diagnose

# 4. 重新启动
npm run dev
```

### 场景 4: 修改智能合约后

```bash
# 1. 编译合约
npm run compile

# 2. 重新部署
npm run deploy

# 3. 导出 ABI
npm run export-abi

# 4. 重启开发服务器
# Ctrl+C 停止
npm run dev
```

### 场景 5: 完全重置项目

```bash
# 警告: 这会删除所有构建文件和依赖
npm run reset

# 然后重新启动
npm run quick-start
```
