# 瀛州纪 - 系统设计说明

## ? 核心理念

《瀛州纪》不是一个传统的战斗游戏，而是一个**链上历史叙事作品**。玩家作为"读者"化身为数字生命，探索一个即将毁灭的数字文明的历史。

### 核心体验

- **不是战斗**，而是**探索与理解**
- **不是装备**，而是**记忆与真相**
- **不是升级打怪**，而是**拼接历史碎片**
- **不是征服世界**，而是**见证文明终结**

---

## ? 系统架构

### 1. 记忆碎片系统 (MemoryFragment.sol)

玩家通过收集记忆碎片来还原历史真相。

#### 碎片类型
- **创世** - 文明起源的记录
- **演化** - 社会发展的痕迹
- **分歧** - 冲突分裂的证据
- **熵化** - 衰败的信号
- **崩溃** - 终结的见证
- **神秘** - 未知的秘密

#### 稀有度
- 普通（白）→ 不凡（绿）→ 稀有（蓝）→ 史诗（紫）→ 传说（金）

#### 核心机制
```solidity
// 碎片可以被组合揭示历史真相
function revealTruth(uint256 recipeId) external {
    // 检查玩家是否拥有所需碎片
    // 揭示完整的历史叙事
    // 记录到世界账本
}
```

**预设碎片：**
1. 创世种子 - `function genesis() { emit WorldCreated(...); }`
2. 第一份合约 - "在区块 #0，第一个智能合约被部署..."
3. 原初对话 - "两个合约首次调用彼此..."
4. 共识诞生 - `mapping(address => bool) public consensus;`
5. 记忆循环 - "合约开始引用自己的历史状态..."
6. 分歧预兆 - `event ConflictDetected(...);`
7. 熵化信号 - "数据开始...损坏...0x████..."
8. 最终交易 - `function finalizeWorld() { ... }`

### 2. 探索者系统 (Explorer.sol)

替代传统的战斗属性，玩家拥有探索相关的能力。

#### 核心属性
- **理解力** (Comprehension) - 从NPC获取深层信息
- **记忆容量** (Memory Capacity) - 可持有的碎片数量
- **洞察力** (Insight) - 发现隐藏线索
- **共鸣度** (Resonance) - 与NPC建立深度联系
- **感知力** (Perception) - 察觉世界熵化

#### 成长系统
```solidity
// 不是战斗经验，而是探索经验
function recordFragmentCollection(address explorer, uint256 fragmentId);
function recordTruthRevealed(address explorer, uint256 recipeId);
function recordNPCEncounter(address explorer, uint256 intimacyLevel, bool isDeep);
```

### 3. 个性化NPC系统 (PersonalizedAINPC.sol)

每个NPC都会根据与玩家的交互形成独特个性。

#### NPC类型
1. **史官** (Archivist) - 记录与历史的守护者
2. **工匠** (Architect) - 创世规则的构建者
3. **商序** (Mercantile) - 资源流动的仲裁者
4. **先知** (Oracle) - 未来可能性的预言者
5. **遗忘** (Entropy) - 混沌与衰变的化身

#### 个性特征（0-100）
- **开放性** (Openness) - 愿意分享信息的程度
- **正式度** (Formality) - 语言风格的正式程度
- **话痨度** (Verbosity) - 回答的详细程度
- **情绪化** (Emotionality) - 情感表达的强度

#### 玩家特定记忆
```solidity
struct PlayerMemory {
    uint256 interactionCount;     // 交互次数
    uint256 intimacyLevel;        // 亲密度（0-100）
    EmotionType currentEmotion;   // 当前情感
    uint256 trustLevel;           // 信任等级
    mapping(bytes32 => uint256) topicFrequency;  // 话题偏好
}
```

- 随着交互增加，亲密度提升
- 根据对话质量，信任度变化

### 4. AI对话系统 (personalizedAI.ts)

根据NPC个性和玩家关系生成独特回答。

#### 系统提示词构建
```typescript
function buildPersonalizedPrompt(context: DialogueContext): string {
    // 基础身份（史官/工匠/先知等）
    // + 个性调整（开放/正式/话痨/情绪化）
    // + 关系状态（亲密度/信任度/情感）
    // + 话题偏好（经常讨论的主题）
    // + 世界状态（熵化程度）
}
```

#### 衰变效果
- 0-30%: 基本正常
- 30-60%: 添加不确定性标记
- 60-90%: 部分文字损坏（█）
- 90-100%: 几乎不可读

#### 示例对话
```
[亲密度: 20, 情感: 中立]
"根据区块链账本的记录，这个问题涉及到区块 #1234 的历史事件..."

[亲密度: 70, 情感: 友善]
"朋友，很高兴再次见到你。让我为你查询... 我们已经交流了很多次，你的问题总是如此深刻。"
```

## ? 核心特色

### 1. 叙事驱动
- 不是传统游戏，而是互动叙事
- 通过碎片拼接理解完整故事
- 代码即诗，技术即叙事

### 2. 个性化AI
- 每个NPC都有独特个性
- 根据交互历史形成记忆
- 情感和信任动态变化

### 3. 链上永存
- 所有历史记录在链上
- 真相揭示不可篡改
- 世界毁灭后仍可考古

### 4. 探索体验
- 强调理解和共鸣
- 重视对话和发现
- 见证而非征服

## ? 设计哲学

> "瀛州纪不是一款游戏，而是一段被智能合约记录下来的文明梦境。"

- **记忆** 替代 装备
- **理解** 替代 战斗
- **共鸣** 替代 征服
- **见证** 替代 拯救

玩家不是英雄，而是**最后的见证者**。
任务不是拯救世界，而是**理解历史**。
结局不是胜利，而是**永恒的记录**。
