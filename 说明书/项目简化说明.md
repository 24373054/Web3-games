# 瀛州纪 - 项目简化说明

## 简化目标

根据用户需求，将游戏简化为核心功能:
1. 创建数字生命NFT
2. 探索历史遗迹的3D世界

移除了所有复杂的小游戏、任务系统等额外功能。

## 主要变更

### 保留的核心功能

? **数字生命系统** (`DigitalBeingCard.tsx`)
- NFT创建和管理
- 探索者身份系统

? **3D遗迹探索** (`ExploreRelic.tsx`)
- 五大纪元的3D世界
- 第一人称探索体验
- 沉浸式叙事

? **智能合约**
- `DigitalBeing.sol` - 数字生命NFT
- `MemoryFragment.sol` - 记忆碎片NFT
- `WorldLedger.sol` - 世界账本

? **自动化脚本**
- `start-dev.js` - 一键启动
- 环境检测
- 依赖安装
- 合约编译
- **合约自动部署** (新增)

### 删除的功能

? 小游戏系统 (MiniGames, MiniGamesHub)
- Memory Sorting (记忆排序)
- Handshake Protocol (握手协议)
- 2048游戏
- OneLine游戏

? 游戏中心 (GameHub)
- 复杂的NPC对话系统
- 关键词触发系统
- 碎片获取小游戏

? 任务系统 (QuestSystem)
- 任务管理器
- 任务进度追踪
- 纪元推进机制

? 市场系统 (FragmentMarketplace)
- NFT交易
- 碎片买卖

? 其他组件
- 纪元总览 (EraOverview)
- 游戏进度 (GameProgress)
- 碎片收藏 (FragmentCollection)
- NPC对话 (NPCDialogue, DialogueInterface)
- 地图探索器 (MapExplorer)
- 世界状态 (WorldStatus)
- 设置面板 (SettingsPanel)
- 游戏结局 (GameEnding, ImprovedGameEnding)

? 辅助库
- questManager.ts
- mapData.ts
- fragmentData.ts
- personalizedAI.ts
- gameData.ts

### 新增功能

? **合约自动部署**
- 启动脚本自动检测合约状态
- 未部署时自动部署到本地网络
- 已部署时复用现有合约地址

## 游戏流程 (简化版)

```
1. 启动游戏
   ↓
2. 连接MetaMask钱包
   ↓
3. 创建数字生命NFT
   ↓
4. 进入3D历史遗迹
   ↓
5. 探索五大纪元世界
```

## 启动脚本增强

### start-dev.js 新功能

```
步骤1: 环境检测 (Node.js, npm)
  ↓
步骤2: 依赖检查与安装
  ↓
步骤3: 智能合约编译
  ↓
步骤4: 智能合约部署 (新增)
  - 检查是否已部署
  - 未部署则自动部署到本地网络
  - 已部署则使用现有地址
  ↓
步骤5: 启动Next.js开发服务器
```

### 使用方式

```bash
# 一键启动 (推荐)
npm run quick-start

# Windows
start.bat

# Linux/Mac
./start.sh
```

## 注意事项

1. **合约部署**: 首次启动需要运行本地Hardhat节点
   ```bash
   npx hardhat node
   ```

2. **MetaMask配置**: 需要手动添加Localhost网络

3. **3D资源**: 确保 `public/yz_3Dtest/` 目录完整

4. **环境变量**: 如需使用AI功能,配置 `.env.local`
