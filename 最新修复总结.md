## ?? 关于 "Internal JSON-RPC error"

### 错误信息

```
MetaMask - RPC Error: Internal JSON-RPC error.
Error: could not coalesce error (error={ "code": -32603, "message": "Internal JSON-RPC error." }
```

### 原因分析

这个错误**不是代码问题**，而是因为：

1. **忘记重置 MetaMask 账户**
   - 重启 Hardhat 后区块链状态重置
   - MetaMask 还记得之前的 nonce
   - 导致 nonce 不匹配

2. **网络连接问题**
   - MetaMask 没有连接到 Localhost 网络
   - 链ID 不是 31337

3. **区块链节点未运行**
   - `npm run game` 没有成功启动

### 解决方案

现在游戏会在首次打开时**自动显示弹窗提示**，告诉用户：

```
┌─────────────────────────────────────────┐
│  M  MetaMask 使用提示                   │
│  每次重启游戏后需要重置 MetaMask 账户   │
├─────────────────────────────────────────┤
│  > 为什么需要重置账户？                 │
│  每次重启 Hardhat 本地区块链时，       │
│  区块链状态会重置，但 MetaMask 还记得   │
│  之前的交易历史（nonce）。             │
│  如果不重置，会导致交易失败。           │
│                                         │
│  * 重置步骤                             │
│  1. 点击 MetaMask 浏览器扩展图标        │
│  2. 点击右上角的"设置"图标              │
│  3. 选择"高级"                          │
│  4. 找到"重置账户"                      │
│  5. 点击并确认                          │
│                                         │
│  + 注意事项                             │
│  ? 重置账户不会删除账户或私钥           │
│  ? 只是清除本地交易历史                 │
│  ? 每次重启游戏后都需要重置一次         │
│  ? 私钥只需导入一次，不用重新导入       │
└─────────────────────────────────────────┘
```

### 用户操作

遇到交易错误时：

```bash

点击浏览器扩展图标

# 2. 重置账户
设置 → 高级 → 重置账户

# 3. 刷新页面
按 F5 刷新游戏页面

# 4. 重试交易
再次点击"创建数字生命"
```

## ? 关键提示

### ? 私钥只需导入一次

Hardhat 每次启动生成的账户都是相同的：
- 地址: `0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266`
- 私钥: `0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80`

导入到 MetaMask 后永久保存，不需要重新导入。

### ?? 每次重启需要重置账户

**必须做！** 否则会出现 "Internal JSON-RPC error"

**操作**:
```
MetaMask → 设置 → 高级 → 重置账户
```

**原理**:
- 重启 Hardhat = 区块链重置 = nonce 归零
- MetaMask 还记得旧的 nonce
- 发送交易时 nonce 不匹配 → 失败
- 重置账户 = 清除 nonce 历史 → 成功
