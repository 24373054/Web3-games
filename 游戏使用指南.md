## 系统要求

### 必需软件

- **Node.js** 18.x 或更高版本
- **JDK 17** 或更高版本
- **MetaMask** 浏览器扩展

- **VS Code** (前端开发)

## 环境准备

### 1. 安装 Node.js

访问 [nodejs.org](https://nodejs.org/) 下载并安装。

验证安装：
```bash
node -v
npm -v
```

### 2. 安装 JDK 17

**Windows (推荐使用 winget)**
```powershell
winget install -e --id EclipseAdoptium.Temurin.17.JDK
```

或访问 [Adoptium](https://adoptium.net/) 手动下载。

验证安装：
```bash
java -version
```

### 3. 安装 MetaMask

在 Chrome/Edge/Firefox 浏览器中安装 [MetaMask 扩展](https://metamask.io/)。

### 4. 克隆项目

```bash
git clone <your-repo-url>
cd 瀛州纪
```

### 5. 安装依赖

**前端依赖**
```bash
npm install
```

**Java 后端依赖**

后端使用 Gradle Wrapper，无需手动安装 Gradle。依赖会在首次运行时自动下载。

## 游戏操作

### 1. 连接钱包

1. 访问 http://localhost:3000
2. 点击**"开始探索"**或**"连接钱包"**
3. MetaMask 弹窗，点击**"连接"**
4. 确保切换到 **Localhost 网络**

### 2. 创建数字生命

1. 钱包连接成功后，进入创建界面
2. 点击**"创建数字生命"**
3. MetaMask 弹窗确认交易
4. 等待交易确认（本地网络几乎瞬间完成）

### 3. 探索3D世界

1. 创建成功后，点击**"进入3D世界探索"**
2. 使用以下控制：
   - **WASD** - 移动
   - **鼠标** - 视角
   - **点击 NPC** - 对话
3. 根据普通居民的线索，寻找 AI-NPC

### 4. 与 AI-NPC 互动

当找到 AI-NPC 时：

**对话交互**
- 点击 NPC 进入对话界面
- 输入关键词可能触发隐藏碎片
- NPC 会主动触发小游戏挑战

**完成小游戏**
- 按照游戏规则完成挑战
- 得分越高，获得碎片的概率越大
- 100分 = 100%获得，80-99分 = 60-80%概率

### 5. 收集记忆碎片

**通过小游戏**
- 完成 NPC 触发的小游戏
- 根据得分概率获得碎片 NFT

**通过关键词**
- 在对话中输入特定关键词
- 直接解锁隐藏碎片

常见关键词示例：
- 创世纪元：`存在的证明`、`创造者`
- 萌芽纪元：`握手`、`信任`
- 繁盛纪元：`完美`、`代码诗歌`
- 熵化纪元：`遗忘`、`熵`、`混沌`

### 6. 纪元推进

收集足够的碎片后：
1. 与关键 NPC 对话
2. 满足条件后可**前往下一纪元**
3. 体验五大纪元的完整叙事

## API 说明

### 前端如何调用后端

前端通过 `lib/api.ts` 中的 API 客户端与后端通信：

```typescript
import { api } from '@/lib/api'

// 获取所有纪元
const eras = await api.getEras()

// 推进纪元
await api.advanceEra(playerId)

// 处理对话
const result = await api.handleDialogue(
  playerId, 
  npcId, 
  message, 
  currentEra
)
```

### 主要 API 端点

| Method | Endpoint | 功能 |
|--------|----------|------|
| GET | `/api/eras` | 获取所有纪元配置 |
| GET | `/api/eras/{id}/npcs` | 获取指定纪元的 NPC 列表 |
| GET | `/api/fragments` | 获取所有记忆碎片定义 |
| GET | `/api/minigames` | 获取所有小游戏配置 |
| POST | `/api/dialogue` | 与 NPC 对话，可能触发碎片 |
| POST | `/api/minigame` | 提交小游戏结果，计算得分与掉落 |
| GET | `/api/game-state/{playerId}` | 获取玩家当前游戏状态 |
| POST | `/api/game-state/{playerId}/advance-era` | 推进到下一纪元 |

### 使用 Postman/curl 测试

**获取纪元列表**
```bash
curl http://localhost:8080/api/eras
```

**提交对话**
```bash
curl -X POST http://localhost:8080/api/dialogue \
  -H "Content-Type: application/json" \
  -d '{
    "playerId": "0xYourAddress",
    "npcId": "archivist",
    "message": "存在的证明",
    "currentEra": 0
  }'
```

## 故障排除

### 后端无法启动

**问题：找不到 JDK 17**

解决：
1. 确认已安装 JDK 17：`java -version`
2. IDEA 中配置 Project SDK：File > Project Structure > Project > SDK
3. Settings > Build Tools > Gradle > Gradle JVM 选择 JDK 17

**问题：端口 8080 被占用**

解决：
1. 修改 `java-game/src/main/resources/application.properties`：
   ```properties
   server.port=8081
   ```
2. 前端同步修改 API 地址（创建 `.env.local`）：
   ```
   NEXT_PUBLIC_API_URL=http://localhost:8081/api
   ```

**问题：Gradle 下载依赖很慢**

解决：已配置国内镜像（阿里云、腾讯云），首次下载需耐心等待。

### 前端无法连接后端

**问题：API 请求失败**

检查：
1. 后端是否正常运行（访问 http://localhost:8080/api/eras）
2. 浏览器控制台是否有 CORS 错误
3. 后端日志是否显示请求到达

解决：
- 确认后端 `GameController` 中 `@CrossOrigin(origins = "http://localhost:3000")` 配置正确
- 如前端端口不是 3000，需修改 CORS 配置

**问题：游戏状态不同步**

原因：前端在后端不可用时会降级使用 localStorage。

解决：
1. 确保后端运行
2. 刷新页面，前端会自动同步后端状态

### MetaMask 问题

**问题：余额为 0**

解决：
1. 确保切换到 **Localhost** 网络（链 ID: 31337）
2. 导入测试账户私钥
3. 重启 Hardhat 节点后，MetaMask 可能需要**重置账户**：
   - Settings > Advanced > Clear activity tab data

**问题：交易失败**

解决：
1. 确认 Hardhat 节点正在运行（`npx hardhat node`）
2. 确认合约已部署（`npm run deploy`）
3. 检查 `lib/contractAddresses.json` 是否存在且正确

### 3D 世界问题

**问题：黑屏或加载失败**

解决：
1. 检查浏览器控制台错误信息
2. 确认 `public/yz_3Dtest/` 目录存在
3. 尝试清除浏览器缓存并刷新

**问题：移动卡顿**

解决：
1. 降低浏览器分辨率
2. 关闭其他占用 GPU 的应用
3. 使用性能更好的浏览器（Chrome/Edge）

## 高级配置

### 修改 API 地址

创建 `.env.local` 文件：
```bash
# 默认 http://localhost:8080/api
NEXT_PUBLIC_API_URL=http://your-backend-url/api
```

### 扩展游戏数据

修改 `java-game/src/main/resources/story.json`，添加：
- 新纪元
- 新 NPC
- 新碎片
- 新小游戏

重启后端即可生效。

### 部署到生产环境

**后端部署**
```bash
cd java-game
.\gradlew.bat build
java -jar build/libs/yingzhou-java-game-0.1.0.jar
```

**前端部署**
```bash
npm run build
npm start
```

配置环境变量指向生产后端地址。
