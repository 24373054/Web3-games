#### 数据来源

1. **打开**: `lib/gameData.ts`
2. **复制**: 所有 `ERAS`、`AI_NPCS`、`MEMORY_FRAGMENTS`、`MINIGAMES` 的数据
3. **转换**: 从 TypeScript 格式转为 JSON 格式
4. **粘贴**: 到 `story.json`

#### 快速方法

可以运行以下脚本自动生成（需要你创建）：

```javascript
// scripts/export-story-json.js
const fs = require('fs');
const { ERAS, AI_NPCS, MEMORY_FRAGMENTS, MINIGAMES } = require('../lib/gameData');

const storyData = {
  eras: ERAS,
  npcs: AI_NPCS,
  fragments: MEMORY_FRAGMENTS,
  minigames: MINIGAMES
};

fs.writeFileSync(
  'java-game/src/main/resources/story.json',
  JSON.stringify(storyData, null, 2)
);

console.log('✓ story.json 生成成功！');
```

运行：
```bash
node scripts/export-story-json.js
```

## 🐛 常见问题

### Q1: story.json 数据不完整怎么办？

**A**: 游戏仍可运行，但：
- 只能探索有数据的纪元
- 部分 NPC 可能缺失
- 部分碎片无法触发

**解决**: 参考 `lib/gameData.ts` 补全数据。

### Q2: 后端启动失败？

**A**: 检查：
1. JDK 17 是否安装：`java -version`
2. 端口 8080 是否被占用
3. 查看后端日志排查错误

### Q3: 前端无法连接后端？

**A**: 
1. 确认后端运行中：访问 `http://localhost:8080/api/eras`
2. 检查 `.env.local` 的 API_URL 配置
3. 查看浏览器控制台错误
