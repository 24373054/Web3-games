# 瀛州纪 - 合约配置说明

## ? 合约配置机制

### 工作原理

瀛州纪使用 **自动化合约配置系统**，完全避免手动配置环境变量的麻烦：

```
一键启动脚本 (start-game.js)
    ↓
1. 启动本地区块链 (Hardhat Node)
    ↓
2. 自动部署所有合约 (deploy.js)
    ↓
3. 保存合约地址到 lib/contractAddresses.json
    ↓
4. 清理 Next.js 缓存（避免读取旧配置）
    ↓
5. 启动开发服务器 (Next.js)
    ↓
6. 前端自动读取 lib/contractAddresses.json
```

### 配置文件位置

- **合约地址文件**: `lib/contractAddresses.json`
- **部署脚本**: `scripts/deploy.js`
- **合约读取**: `lib/contracts.ts`

### ?? 重要：不要使用 .env.local

本项目 **不需要** 创建 `.env.local` 文件！

所有配置都通过 `lib/contractAddresses.json` 自动管理。

## ? 使用方法

### 方法1：一键启动（推荐）

```bash
# Windows
一键启动.bat

# 或
npm run game
```

这个脚本会：
- ? 检查依赖
- ? 启动本地区块链
- ? 自动部署合约
- ? 保存合约地址
- ? 清理缓存
- ? 启动游戏

### 方法2：分步启动

如果需要手动控制每个步骤：

```bash
# 终端1: 启动区块链节点
npx hardhat node

# 终端2: 部署合约
npx hardhat run scripts/deploy.js --network localhost

# 终端3: 启动游戏
npm run dev
```

## ? 排查问题

### 问题1: "合约地址未配置"

**原因**: `lib/contractAddresses.json` 文件不存在或为空

**解决**:
```bash
# 重新部署合约
npx hardhat run scripts/deploy.js --network localhost
```

部署成功后，会在 `lib/contractAddresses.json` 中看到：
```json
{
  "chainId": 31337,
  "rpcUrl": "http://127.0.0.1:8545",
  "worldLedger": "0x5FbDB2315678afecb367f032d93F642f64180aa3",
  "digitalBeing": "0xe7f1725E7734CE288F8367e1Bb143E90bb3F0512",
  ...
}
```

### 问题2: "合约调用失败"

**可能原因**:
1. 区块链节点未运行
2. 使用了缓存的旧配置
3. MetaMask 连接错误的网络

**解决步骤**:

1?? **检查区块链节点是否运行**:
```bash
curl http://localhost:8545
```
应该返回 405 错误（正常，说明节点在运行）

2?? **清理缓存并重启**:
```bash
# 停止所有服务 (Ctrl+C)

# 删除缓存
rm -rf .next
rm -rf node_modules/.cache

# 重新启动
npm run game
```

3?? **检查 MetaMask 配置**:
- 网络名称: `Localhost` 或 `Hardhat`
- RPC URL: `http://127.0.0.1:8545`
- 链ID: `31337`
- 货币符号: `ETH`

4?? **导入测试账户**:

运行 `npx hardhat node` 时，终端会显示测试账户私钥：
```
Account #0: 0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266 (10000 ETH)
Private Key: 0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80
```

在 MetaMask 中导入这个私钥。

### 问题3: "缓存问题"

**症状**: 修改了合约地址，但前端还在使用旧地址

**原因**: Next.js 编译缓存

**解决**:
```bash
# 删除 .next 目录
rm -rf .next

# 或使用一键启动（自动清理缓存）
npm run game
```

### 问题4: "合约部署后仍然报错"

**检查清单**:

1. ? 确认 `lib/contractAddresses.json` 存在且包含有效地址
2. ? 确认区块链节点正在运行 (`http://localhost:8545`)
3. ? 确认 MetaMask 连接到正确网络 (31337)
4. ? 确认使用的账户有足够 ETH
5. ? 确认已清理 Next.js 缓存
6. ? 确认浏览器控制台没有其他错误

## ?? 高级配置

### 自定义RPC地址

如果需要使用不同的区块链节点：

1. 修改 `hardhat.config.js`:
```javascript
networks: {
  localhost: {
    url: "http://127.0.0.1:8545"  // 修改这里
  }
}
```

2. 修改 `lib/provider.ts` 中的默认RPC:
```typescript
const LOCAL_RPC_URL = 'http://127.0.0.1:8545'  // 修改这里
```

3. 重新部署合约

### 切换到测试网或主网

**不推荐！** 本游戏设计为本地游戏，需要频繁铸造NFT和交互。

如果确实需要：

1. 在 `hardhat.config.js` 添加网络配置
2. 部署到目标网络：
```bash
npx hardhat run scripts/deploy.js --network goerli
```
3. 手动修改 `lib/contractAddresses.json`

6. ? **不要修改 contractAddresses.json**（由脚本自动管理）
